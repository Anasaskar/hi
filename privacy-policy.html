7.4 Automated Data Deletion Callback (For Platforms)

CloyAI supports automated data deletion requests from platforms such as Facebook. When a user requests data deletion through Facebook, we receive a callback and process the deletion automatically.

Deletion Callback Endpoint: POST /api/facebook/data-deletion

Response Format: We respond with a JSON object containing a status URL and confirmation code:

{
  "url": "https://cloyai.com/deletion-status?id=<CONFIRMATION_CODE>",
  "confirmation_code": "<USER_ID_OR_UNIQUE_TOKEN>"
}

Users can visit the status URL to check the progress of their deletion request. The confirmation code can be used to verify the deletion status.

Appendix: Facebook Data Deletion Callback - Technical Specification

This section provides technical details for developers implementing the Facebook data deletion callback.

Endpoint: POST /api/facebook/data-deletion

Request from Facebook: Facebook will send a POST request with a signed request parameter containing the user's ID.

Expected Response (JSON):

{
  "url": "https://cloyai.com/deletion-status?id=abc123xyz",
  "confirmation_code": "abc123xyz"
}

Field Descriptions:

  • url: A link where the user can check the status of their deletion request
  • confirmation_code: A unique identifier for this deletion request (can be the user ID, a UUID, or a time-stamped token)

Implementation Notes:

  • Verify the signed request from Facebook using your App Secret
  • Extract the user ID from the signed request
  • Queue the deletion request in your system
  • Generate a unique confirmation code
  • Return the JSON response immediately (don't wait for deletion to complete)
  • Process the actual deletion asynchronously
  • Update the status URL so users can check progress

For more information, see Facebook's documentation: Data Deletion Callback
